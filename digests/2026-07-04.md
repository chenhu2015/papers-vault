OPID (0.88) is the structural standout. It closes the UCOB (Jul 3) gap that remained open: UCOB proposed context-view credit as a new 7th axis of the credit assignment hierarchy but required pre-built external skill memory — a practical constraint that limits deployment when skill memory is unavailable or distribution-mismatched. OPID's on-policy extraction makes context-view credit practical: it generates its own skill context from completed rollouts in the same training batch, meaning the skill supervision is always matched to the current policy's state distribution. The log-probability shift signal is also more precise than UCOB's return-to-go comparison — it localizes credit to individual tokens rather than trajectory segments. OPID thus both validates and sharpens the 7th axis.

SPARC (0.78) is the most structurally novel VLM paper in the digest. The two-stage perception-then-reasoning architecture is the VLM counterpart to PASS's (Jul 1) channel separation insight: where PASS showed that mixing process and outcome signals in the same GRPO update creates channel contamination, SPARC shows that mixing perception and reasoning in the same VLM forward pass creates analogous token-context contamination under distribution shift. The 200× token budget reduction and OOD improvement together make SPARC the strongest practical result in the VLM test-time scaling thread to date.

ReasonRAG (0.75) connects two prior threads in a single paper: it applies TRIAGE's (Jul 2) role-typed credit intuition (different action roles deserve different credit treatment) to the RAG sub-task domain (query/evidence/answer), and it quantifies the practical cost of PASS's (Jul 1) sparse-reward diagnosis in concrete numbers (18× more data needed for outcome-only vs. process-supervised). This makes ReasonRAG the most data-efficiency-focused entry in the process reward thread.

MemBuilder (0.75) extends the credit assignment hierarchy to a new domain: long-term memory construction. Its contribution-aware gradient weighting is structurally similar to TRIAGE's role-typed credit but applied to memory component dimensions rather than agentic action types — it answers "which memory write mattered" rather than "which navigation action mattered." The 4B-beats-closed-source result is notable because it suggests attribution quality can substitute for model scale in memory-augmented dialogue.

VisCritic (0.73) fills the VLM process reward gap that has persisted through the entire credit assignment thread: all prior PRM papers in the vault used text-only verification, and the GUI domain requires visual state change as the primary verification signal. The Siamese pre/post-action comparison is the GUI analog of TAPO's (Jul 3) parameter-determinism counterfactual — both papers make the same argument (the standard text/outcome signal systemically misassigns credit in their domain) using domain-appropriate witnesses.

**Updated credit assignment framing:** the context-view axis (axis 7, introduced by UCOB Jul 3, sharpened by OPID today) now has a practical on-policy instantiation. The domain coverage of the credit assignment thread has expanded from math/code text RL → agentic navigation text RL → VLM GUI agents (VisCritic) and RAG agents (ReasonRAG), with memory construction as a newly opened domain (MemBuilder).
