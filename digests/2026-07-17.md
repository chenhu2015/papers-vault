RIPO (0.90) is today's most significant paper. The insight is deceptively simple but foundational: every RL algorithm that uses PPO-style clipping is implicitly assuming policy distances live in Euclidean space, but policy distributions are points on a Riemannian manifold where the natural metric is the Fisher information metric. Euclidean clipping is geometrically wrong — it creates conservative-in-low-probability / aggressive-in-high-probability asymmetry that collapses exploration. RIPO's isometric fix achieves 60% improvement over GRPO on AIME24. This is not a data or curriculum fix; it's an algorithm-level fix that applies to every GRPO variant in the vault.

DeReason (0.82) provides the clearest empirical evidence for the SFT→RL two-stage pattern that the vault has observed across many papers (TTI, RSPO, VRRL, Learning When to Plan, TurnOPD). The key new finding: the split is not just helpful, it's principled — RL is sample-inefficient on STEM base models because they lack domain knowledge foundations, but SFT→RL compounds because SFT establishes the representation substrate that RL exploration needs. Difficulty-based data routing operationalizes this: SFT gets breadth, RL gets depth.

EGRSD (0.80) partially addresses the persistent vault gap of "unified-loss reasoning + self-verification." The entropy confidence gate is exactly the mechanism that prior searches for this thread failed to find: instead of a binary gate (supervised or RL), EGRSD uses teacher entropy as a soft gate that continuously adjusts token-level update weights. CL-EGRSD's causal-lookahead adds the temporal dimension — transient high-entropy positions (flanked by low-entropy context) are still informative and should not be down-weighted. This is the closest the vault has come to resolving the unified-loss gap.

TurnOPD (0.78) and Prune-OPD (0.76) together form a coherent pair addressing the same problem — compute waste in long-horizon OPD — from complementary granularities. TurnOPD wastes turns (turn-level truncation via probe statistics); Prune-OPD wastes tokens within turns (token-level truncation via prefix-drift detection). Both confirm the generalized insight from RSPO and Learning When to Plan: indiscriminate dense supervision across all steps/tokens degrades long-horizon training, and selective supervision allocation is the fix.

**Emerging thread — RL optimization internals (new):** Three papers now characterize algorithmic improvements to the RL training loop itself: RIPO (geometric fix for PPO-Clip), EGRSD (entropy-gated token weighting for RL+OPD), and earlier vault papers on GRPO variants (ARMOR's off-policy anchor, RSPO's dense-for-sampling design). The unified insight: the default uniform treatment of all updates — all regions of policy space, all token positions, all trajectory segments — is wrong, and principled non-uniform weighting consistently improves performance. RIPO fixes policy-space geometry, EGRSD fixes token-level weighting, Prune-OPD + TurnOPD fix trajectory-level compute allocation.
