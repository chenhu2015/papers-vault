TTI (0.82) is today's most significant paper. The insight is that test-time scaling for agents has been studied along the wrong axis — per-step inference tokens — while the natural unit for agent compute is *interaction steps* in the environment. Curriculum RL that gradually lengthens rollout horizons teaches the model to allocate environmental compute (exploration, backtracking, re-planning) rather than just reasoning tokens. SOTA on WebVoyager and WebArena from a 12B model is the empirical payoff. This directly extends the vault's RL training thread (RSPO, ARMOR, DISA) to the agentic TTS setting.

Scaling TTS for Agents (0.78) confirms two vault themes in the agent setting: (1) list-wise diversity beats point-wise selection — consistent with LLMs-as-a-Jury; (2) indiscriminate reflection hurts — consistent with RSPO's lesson that dense rewards at every step degrade optimization. The "when to reflect" finding also anticipates Learning When to Plan.

ViCrit (0.75) addresses a gap that has been implicit in the VLM RL thread: most VLM RL proxy tasks either use weak rewards (model-based judges with calibration issues, per Multimodal Reward Hacking) or require complex setups. ViCrit's binary exact-match hallucination-localization reward is unambiguous and verifiable in the same way math's final-answer reward is. The cross-domain transfer (to abstract visual reasoning and visual math) is the key validation.

General AgentBench (0.75) provides the diagnostic counterpoint to TTI's positive result: why naive TTS fails for general agents. Context ceiling (sequential) and verification gap (parallel) are the two structural barriers. The verification gap is precisely the mechanism that LLMs-as-a-Jury identified as the limit of consensus in the math setting — shared-error floor — now extended to the agent execution domain.

Learning When to Plan (0.74) ties the day's theme together: dynamic compute allocation (when to plan) via RL is a general principle that appears across math reflection (RSPO), agent reflection (Scaling TTS for Agents), and agent planning (this paper). The SFT→RL two-stage design is now a vault-wide pattern across agentic RL papers.

**Emerging thread — agentic TTS (new):** Five papers now characterize test-time compute scaling for agents: TTI (interaction horizon curriculum RL), Scaling TTS for Agents (reflection timing and diversity), General AgentBench (failure modes: context ceiling, verification gap), Learning When to Plan (dynamic planning allocation via RL), and General AgentBench's benchmark. The unified insight: agentic TTS requires training-time RL to learn *selective* compute allocation (when to interact more, when to reflect, when to plan), not just inference-time heuristics. Static TTS policies uniformly degrade long-horizon agent performance.
